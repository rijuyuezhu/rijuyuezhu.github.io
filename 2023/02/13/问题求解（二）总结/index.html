Week 1

Concepts

TC 1 The Role of Algorithms in Computing

1.1 Algorithms

  • Algorithm is any well-defined computational procedure that taks some value/set of values as input and produces some value/set of values as output.
  • Algorithm is a tool for solving a well-specified computational problem(specify the desired input/output relationship in general terms).
  • Sorting problem:
    • Given a seqence of \(n\) numbers \(\langle a_1, a_2, \cdots, a_n\rangle\).
    • Output a permutation of it, \(\langle a_1', a_2', \cdots, a_n'\rangle\) such that \(a_1'\le a_2'\le \cdots, a_n'\).
  • Instance of a problem: A specific input(satisfy the constraints in the statement).
  • Correctness of an algorithm: 1. correct output; 2. always halts for every legal input instance. A correct algorithm solves the given computational problem.
  • Characteristics that are common to many algorithmic problems:
    • Have many candidate solutions. But we want to find the "best".
    • Have practical applications.
  • Data Structure: A way to store and organize data in order to facilitate access and modifications(strengths and limitations).
  • Hard problem, e.g. NP-complete problems: A class of problems that
    • An efficient algorithm hasn't been found.
    • Equivalent to each other.
    • Some are similar to ordinary problems.
    • e.g. traveling-salesman problem (TSP).
  • Parallelism
    • Fact: physical limitations let processor clock speeds' increasing slow down.
    • Chips have several processing "cores" \(\to\) parallel computer.
    • Algorithms with parallelism in mind, multithreaded algorithms.

1.2 Algorithm as a technology

  • Computing time and space in memory are bounded resource. So we need efficiency.
  • Different algorithms devised to solve the same problem often differ dramatically in their efficiency. e.g. insertion sort and merge sort.
  • Algorithms are a technology, just like other computer technologies. Everything needs algorithms.

TC 2 Getting Started

2.1 Insertion sort

  • Definition of sorting problem(See TC 1)
  • Pseudocode: clear and concise; ignore issues of software engineering.
  • Insertion sort. P18.
  • Loop invariants: show why a algorithm is correct.(Initialization, Maintenance, Termination).
    • The first two just like mathematical induction.
    • The third one shows the correctness of the algorithm.

2.2 Analyzing algorithms

  • Analyzing an algorithm: predict the resouces that the algorithm requires. (memory, communication bandwidth, computer hardware, but computational time is the most important)
  • RAM model.
    • One after another;
    • Arithmetic, data movement, control takes a constant amount of time.
    • Data type: integer and floating point.
    • ...
  • Analysis of insertion sort.
  • Input size: number or items in the input, total number of bits, ..., one or more.
  • Running time of a algorithm: the number of primitive operations("steps"), a function of input size (such as \(n\)).
    • linear function, quadratic function.
  • Worst-case analysis is more important:
    • As upper bound of the running time.
    • For some algorithms, worst case occurs fairly often.
    • "Average case" is often roughly as bad as the worst case.
  • Average-case running time: probabilistic analysis.
  • Randomized algorithm, expected running time.
  • Order of growth(rate of growth).
    • Consider only the leading term of a formula, and ignore the constant coefficient.
    • Like the insertion sort, we write \(\Theta(n^2)\).
    • Generally, one algorithm is more efficient than another if its worst-case running time has a lower order of growth.

2.3 Designing algorithms

  • The incremental approach, e.g. insertion sort.
  • The divide-and-conquer apporach, e.g. merge sort.
  • Meaning:
    1. Divide Break the problem into several subproblems that are similar to the original problem but smaller in size.
    2. Conquer Solve the subproblems recursively. If the size is small enough, solve directly.
    3. Combine Combine these solutions to create a solution to the original problem.
  • Merge procedure and its analysis.
  • Merge sort.
  • Analyzing divide-and-conquer algorithms. Describe its running time by a recurrence equation/recurrence. \[ T(n)= \begin{cases} \Theta(1)&\textnormal{if}\ n\le c, \\ aT(n/b)+D(n)+C(n)&\textnormal{otherwise}. \end{cases} \]

TC 3 Growth of Functions

3.1 Asymptotic notation

  • Asymptotic notation: be applied to functions, not only running time function, but also some other functions.
  • \(\Theta\)-notation means: \(\Theta(g(n))=\lbrace f(n):\) there exist positive constants \(c_1, c_2\) and \(n_0\) such that \(0\le c_1g(n)\le f(n)\le c_2g(n)\) for all \(n>n_0\) \(\rbrace\).
    • We write \(f(n)\in \Theta(g(n))\), or \(f(n)=\Theta(g(n))\).
    • \(g(n)\) is an asymptotically tight bound for \(f(n)\).
    • Every \(f(n)\) need to be asymptotically nonnegative, and so is \(g(n)\).
    • Lower-order terms of an asympotically positve function can be ignored in determining.
  • \(O\)-notation means: \(O(g(n))=\lbrace f(n):\) there exist positive constants \(c\) and \(n_0\) such that \(0\le f(n)\le cg(n)\) for all \(n>n_0\) \(\rbrace\).
    • \(g(n)\) is an asymptotic upper bound.
    • \(\Theta\)-notation is a stronger notation than \(O\)-notation, \(\Theta(g(n))\subseteq O(g(n))\).
    • \(O(g(n))\) gives a running time bound on every input when \(\Theta(g(n))\) for worst-case input.
  • \(\Omega\)-notation means: \(\Omega(g(n))=\lbrace f(n):\) there exist positive constants \(c\) and \(n_0\) such that \(0\le cg(n) \le f(n)\) for all \(n>n_0\) \(\rbrace\).
    • \(g(n)\) is an asymptotic lower bound.
    • Also, \(\Theta(g(n))\subseteq \Omega(g(n))\).
    • \(\Omega(g(n))\) gives a running time bound on every input when \(\Theta(g(n))\) for best-case input.
  • Theorem 3.1: For any two functions \(f(n)\) and \(g(n)\), we have \(f(n)=\Theta(g(n))\) if and only if \(f(n)=O(g(n))\) and \(f(n)=\Omega(g(n))\).
    • In practice, to prove asymptotically tight bounds.(\(\Leftarrow\))
  • Asymptotic notation in equations and inequalities(P49).
  • \(o\)-notation means: \(o(g(n))=\lbrace f(n):\) for any positive constants \(c>0\), there exists a constant \(n_0\) such that \(0\le f(n)< cg(n)\) for all \(n>n_0\) \(\rbrace\).
    • \(g(n)\) is an asymptotic upper bound but not asymptotically tight.
    • \(\displaystyle\lim_{n\to \infty}\dfrac{f(n)}{g(n)}=0\).
  • \(\omega\)-notation means: \(\omega(g(n))=\lbrace f(n):\) for any positive constants \(c>0\), there exists a constant \(n_0\) such that \(0 \le cg(n)< f(n)\) for all \(n>n_0\) \(\rbrace\).
    • \(g(n)\) is an asymptotic lower bound but not asymptotically tight.
    • \(\displaystyle\lim_{n\to \infty}\dfrac{f(n)}{g(n)}=\infty\).
  • Transitivity, Reflexivity, Symmetry, Transpose symmetry.
  • Asymptotically smaller/bigger.
  • Trichotomy is not applied to asymptotical notations.
    • E.g. \(f(n)=n, g(n)=n^{1+\sin n}\), neither \(f(n)=O(g(n))\) nor \(f(n)=\Omega(g(n))\).

3.2 Standard notations and common functions

  • Floors and ceilings. If \(x\in \mathbb R_+, a, b\in \mathbb Z_+\) \[ \begin{gather*} x-1\le \lfloor x\rfloor \le x \le \lceil x\rceil \le x+1. \\ \lceil n/2\rceil + \lfloor n/2 \rfloor =n. \\ \left\lceil \dfrac{\lceil x/a \rceil}{b}\right\rceil=\left\lceil \dfrac{x}{ab}\right\rceil. \\ \left\lfloor \dfrac{\lfloor x/a \rfloor}{b}\right\rfloor=\left\lfloor \dfrac{x}{ab}\right\rfloor. \\ \left\lceil\dfrac{a}{b}\right\rceil\le \dfrac{a+(b-1)}{b}. \\ \left\lfloor\dfrac{a}{b}\right\rfloor\le \dfrac{a-(b-1)}{b}. \end{gather*} \]
  • \(f(n)\) is polynomially bounded if \(f(n)=O(n^k)\) for some constant \(k\).
  • \(f(n)\) is polylogarihmically bounded if \(f(n)=O(\lg^kn)\) for some constant \(k\).
  • Factorials \[ \begin{gather*} n!=\sqrt{2\pi n}\left(\dfrac{n}{e}\right)^n\left(1+\Theta\left(\dfrac{1}{n}\right)\right). \\ n!=o(n^n). \\ n!=\omega(2^n). \\ \lg(n!)=\Theta(n\lg n). \\ n!=\sqrt{2\pi n}\left(\dfrac{n}{e}\right)^ne^{\alpha_n}. \\ \textnormal{where}\ \dfrac{1}{12n+1}<\alpha_n<\dfrac{1}{12n} \end{gather*} \]
  • Function iteration \[ f^{(i)}(n)= \begin{cases} n, & \textnormal{if}\ i=0, \\ f(f^{(i-1)}(n)), & \textnormal{if}\ i>0. \end{cases} \]
  • The iterated logaithm function \[ \lg^*(n)=\min\lbrace i\ge 0: \lg^{(i)} n\le 1\rbrace. \]
  • Fibonacci numbers
    • Definition: \[ F_0=0, F_1=1, Fi=F_{i-1}+F_{i-2}(i\ge 2). \]
    • golden ratio: \[ \begin{gather*} \phi = \dfrac{1+\sqrt 5}{2}, \hat \phi=\dfrac{1-\sqrt 5}{2}. \\ F_i=\dfrac{\phi-\hat\phi}{\sqrt 5}. \end{gather*} \]
    • For \(|\hat\phi|<1\), we have \(\dfrac{|\hat\phi|}{\sqrt 5}<\dfrac{1}{\sqrt 5}<\dfrac{1}{2}\), so \(F_i=\left\lfloor\dfrac{\phi^i}{\sqrt 5}+\dfrac{1}{2}\right\rfloor=\operatorname{round}\left(\dfrac{\phi^i}{\sqrt 5}\right)\).
